# IPCONFIG - stepper-ninja सीरियल टर्मिनल कॉन्फ़िगरेशन

## अवलोकन

`serial_terminal.c` में `ipconfig` कार्यक्षमता stepper-ninja डिवाइस को कॉन्फ़िगर करने के लिए एक सीरियल टर्मिनल इंटरफ़ेस प्रदान करती है। यह उपयोगकर्ताओं को नेटवर्क पैरामीटर (IP, MAC, पोर्ट आदि) सेट करने, टाइमआउट प्रबंधित करने, कॉन्फ़िगरेशन को फ्लैश में सहेजने और डिवाइस को रीसेट करने की सुविधा देती है। कमांड सीरियल कनेक्शन के माध्यम से दर्ज किए जाते हैं, वास्तविक समय में प्रोसेस किए जाते हैं और सही फ़ॉर्मेटिंग के लिए सत्यापित किए जाते हैं।

## पूर्वापेक्षाएँ

- एक सीरियल टर्मिनल एमुलेटर (जैसे Linux पर **minicom**) जो डिवाइस के सीरियल पोर्ट (जैसे `/dev/ttyACM0`) से उचित बॉड रेट (आमतौर पर 115200 8N1) पर कनेक्ट हो।
- Linux पर सीरियल पोर्ट एक्सेस के लिए उपयोगकर्ता को `dialout` समूह का सदस्य होना चाहिए:

  ```bash
  sudo usermod -a -G dialout $USER
  ```

- डिवाइस चालू होना चाहिए और डिवाइस प्रोग्रामिंग के लिए उपयोग किए जाने वाले USB केबल से जुड़ा होना चाहिए।

## सीरियल टर्मिनल सेटअप (Linux)

1. minicom इंस्टॉल करें:

   ```bash
   sudo apt update
   sudo apt install minicom
   ```

2. उचित सीरियल पोर्ट (जैसे `/dev/ttyACM0`) के लिए minicom शुरू करें:

   ```bash
   minicom -D /dev/ttyACM0 -b 115200
   ```

3. minicom में हार्डवेयर/सॉफ्टवेयर फ्लो कंट्रोल अक्षम करें (`Ctrl+A`, फिर `O`, **Serial port setup** चुनें, **F** और **G** को `No` पर सेट करें)।

## समर्थित कमांड

सीरियल टर्मिनल टर्मिनल में टाइप किए गए कमांड स्वीकार करता है, जो कैरेज रिटर्न (`\r`, Enter कुंजी) द्वारा समाप्त होते हैं। कमांड केस-संवेदनशील हैं और नीचे वर्णित सटीक सिंटैक्स का पालन करना चाहिए। अमान्य कमांड या गलत फ़ॉर्मेट एक त्रुटि संदेश लौटाते हैं।

| कमांड | विवरण | उदाहरण | आउटपुट/प्रभाव |
|-------|-------|--------|---------------|
| `help` | उपलब्ध कमांड और उनके विवरण की सूची प्रदर्शित करता है। | `help` | सहायता मेनू प्रिंट करता है। |
| `check` | वर्तमान कॉन्फ़िगरेशन दिखाता है, जिसमें MAC, IP, सबनेट, गेटवे, DNS, DHCP स्थिति, पोर्ट, PHY स्थिति (डुप्लेक्स/गति) और टाइमआउट शामिल हैं। | `check` | विस्तृत विवरण प्रिंट करता है, जैसे `IP: 192.168.1.100`। |
| `ip <x.x.x.x>` | डिवाइस का IP पता निर्दिष्ट मान पर सेट करता है। | `ip 192.168.1.100` | IP अपडेट करता है। `IP changed to 192.168.1.100` प्रिंट करता है। |
| `ip` | वर्तमान IP पता दिखाता है। | `ip` | `IP: 192.168.1.100` प्रिंट करता है। |
| `port <port>` | डिवाइस का पोर्ट नंबर सेट करता है। | `port 5000` | पोर्ट अपडेट करता है। `Port changed to 5000` प्रिंट करता है। |
| `port` | वर्तमान पोर्ट नंबर दिखाता है। | `port` | `Port: 5000` प्रिंट करता है। |
| `mac <xx:xx:xx:xx:xx:xx>` | डिवाइस का MAC पता सेट करता है (हेक्साडेसिमल, कोलन से अलग)। | `mac 00:1A:2B:3C:4D:5E` | MAC अपडेट करता है। `MAC changed to 00:1A:2B:3C:4D:5E` प्रिंट करता है। |
| `mac` | वर्तमान MAC पता दिखाता है। | `mac` | `MAC: 00:1A:2B:3C:4D:5E` प्रिंट करता है। |
| `timeout <value>` | टाइमआउट मान माइक्रोसेकंड में सेट करता है। | `timeout 1000000` | टाइमआउट अपडेट करता है। `Timeout changed to 1000000` प्रिंट करता है। |
| `timeout` | वर्तमान टाइमआउट मान दिखाता है। | `timeout` | `Timeout: 1000000` प्रिंट करता है। |
| `tim <value>` | समय स्थिरांक मान सेट करता है। | `tim 500` | समय स्थिरांक अपडेट करता है। `Time const changed to 500` प्रिंट करता है। |
| `defaults` | डिफ़ॉल्ट कॉन्फ़िगरेशन पुनर्स्थापित करता है। | `defaults` | कॉन्फ़िगरेशन को फ़ैक्टरी डिफ़ॉल्ट पर रीसेट करता है। |
| `reset` | वॉचडॉग टाइमर का उपयोग करके डिवाइस रीसेट करता है। | `reset` | डिवाइस रीसेट शुरू करता है। |
| `reboot` | `reset` के समान, डिवाइस रीबूट करता है। | `reboot` | डिवाइस रीसेट शुरू करता है। |
| `save` | वर्तमान कॉन्फ़िगरेशन को फ्लैश मेमोरी में सहेजता है। | `save` | फ्लैश में कॉन्फ़िगरेशन परिवर्तन सहेजता है। |

### नोट्स

- पैरामीटर वाले कमांड (जैसे `ip <x.x.x.x>`) के लिए सटीक फ़ॉर्मेटिंग आवश्यक है। उदाहरण के लिए, IP पते चार दशमलव संख्याएँ होनी चाहिए जो डॉट से अलग हों (0-255), और MAC पते छह हेक्साडेसिमल बाइट होने चाहिए जो कोलन से अलग हों।
- अमान्य फ़ॉर्मेट एक त्रुटि संदेश उत्पन्न करते हैं, जैसे `Invalid IP format` या `Invalid MAC format`।
- सेटिंग्स बदलने के बाद (जैसे IP, पोर्ट, MAC) फ्लैश में परिवर्तन सहेजने के लिए `save` कमांड जारी करना होगा। अन्यथा, रीसेट पर परिवर्तन खो जाएंगे।
- टर्मिनल `timeout_error` स्थिति के आधार पर लॉक/अनलॉक हो सकता है:
  - यदि `timeout_error == 0` है, तो टर्मिनल लॉक होता है (`Terminal locked.`)।
  - यदि `timeout_error == 1` है, तो टर्मिनल अनलॉक होता है (`Terminal unlocked.`)।

## उपयोग उदाहरण

1. minicom का उपयोग करके डिवाइस से कनेक्ट करें:

   ```bash
   minicom -D /dev/ttyACM0 -b 115200
   ```

2. वर्तमान कॉन्फ़िगरेशन जांचें:

   ```terminal
   check
   ```

   आउटपुट:

   ```terminal
   Current configuration:
   MAC: 00:1A:2B:3C:4D:5E
   IP: 192.168.1.100
   Subnet: 255.255.255.0
   Gateway: 192.168.1.1
   DNS: 8.8.8.8
   DHCP: 1 (1-Static, 2-Dynamic)
   PORT: 5000
   *******************PHY status**************
   PHY Duplex: Full
   PHY Speed: 100Mbps
   *******************************************
   Timeout: 1000000
   Ready.
   ```

3. नया IP पता सेट करें:

   ```terminal
   ip 192.168.1.200
   ```

   आउटपुट: `IP changed to 192.168.1.200`

4. कॉन्फ़िगरेशन सहेजें:

   ```terminal
   save
   ```

5. डिवाइस रीबूट करें:

   ```terminal
   reset
   ```

## तकनीकी विवरण

- **इनपुट हैंडलिंग**: `handle_serial_input` फ़ंक्शन `getchar_timeout_us` के माध्यम से गैर-ब्लॉकिंग तरीके से वर्ण पढ़ता है। यह नॉन-प्रिंटेबल ASCII वर्णों (`\r` को छोड़कर) को अनदेखा करता है और वैध इनपुट को 64-बाइट बफ़र में संग्रहीत करता है। जब `\r` प्राप्त होता है या बफ़र भर जाता है, तो कमांड प्रोसेस होता है और बफ़र साफ़ हो जाता है।
- **कमांड प्रोसेसिंग**: `process_command` फ़ंक्शन `strcmp` और `strncmp` का उपयोग करके कमांड पार्स करता है। पैरामीटर वाले कमांड के लिए, `sscanf` इनपुट फ़ॉर्मेट (जैसे IP, MAC, पोर्ट) सत्यापित करता है। परिवर्तन ग्लोबल वेरिएबल्स (`net_info`, `port`, `TIMEOUT_US`, `time_constant`) पर लागू होते हैं और `save_configuration` के माध्यम से सहेजे जाते हैं।
- **कॉन्फ़िगरेशन स्टोरेज**: `save_configuration` फ़ंक्शन सेटिंग्स को `flash_config` संरचना में कॉपी करता है, जिसे `save_config_to_flash` का उपयोग करके फ्लैश में सहेजा जाता है। `load_configuration` फ़ंक्शन स्टार्टअप पर फ्लैश से सेटिंग्स प्रारंभ करता है।
- **लॉकिंग मैकेनिज्म**: टर्मिनल `timeout_error == 0` होने पर लॉक होता है और `timeout_error == 1` होने पर अनलॉक होता है, जो सीरियल इंटरफ़ेस तक पहुँच को नियंत्रित करता है।

## समस्या निवारण

- **डिवाइस से कोई प्रतिक्रिया नहीं**:
  - सही सीरियल पोर्ट सत्यापित करें (जैसे `ls /dev/tty*` से पोर्ट सूचीबद्ध करें)।
  - सुनिश्चित करें कि बॉड रेट सही है (डिफ़ॉल्ट: 115200)।
  - सीरियल केबल कनेक्शन और डिवाइस की पॉवर जांचें।
- **सीरियल पोर्ट पर अनुमति अस्वीकृत**:
  - उपयोगकर्ता को `dialout` समूह में जोड़ें (पूर्वापेक्षाएँ देखें)।
- **गड़बड़ आउटपुट**:
  - minicom में बॉड रेट, पैरिटी (8N1) और फ्लो कंट्रोल सेटिंग्स सत्यापित करें।
- **अमान्य फ़ॉर्मेट त्रुटियाँ**:
  - सटीक कमांड सिंटैक्स सुनिश्चित करें (जैसे `ip 192.168.1.100`, न कि `ip 192.168.001.100`)।
- **परिवर्तन सहेजे नहीं गए**:
  - सेटिंग्स बदलने के बाद `save` कमांड जारी करें।

## सीमाएँ

- टर्मिनल बफ़र 63 वर्णों तक सीमित है (नल टर्मिनेटर सहित)। इससे लंबे कमांड काट दिए जाएंगे।
- सीरियल इंटरफ़ेस गैर-ब्लॉकिंग है, इसलिए डिवाइस व्यस्त होने पर तेज़ इनपुट छूट सकता है।
- लॉकिंग मैकेनिज्म (जो `timeout_error` पर आधारित है) नेटवर्क स्थितियों के कारण टाइमआउट होने पर अप्रत्याशित रूप से पहुँच प्रतिबंधित कर सकता है।

## संबंधित फ़ाइलें

- `serial_terminal.c`: सीरियल टर्मिनल इंटरफ़ेस और कमांड प्रोसेसिंग लागू करता है।
- `config.h`: `configuration_t` संरचना और `save_config_to_flash`, `load_config_from_flash`, और `restore_default_config` जैसे फ़ंक्शन परिभाषित करता है।
- `wizchip_conf.h`: नेटवर्क कॉन्फ़िगरेशन के लिए `wiz_NetInfo` संरचना और `getPHYSR` जैसे फ़ंक्शन परिभाषित करता है।
